## Governance

- Extend an existing product doc before creating a new one.
- Update product docs when user-visible behavior, ranking, search, feed, discovery, onboarding, retention, or feature inventory changes.
- Keep product docs concise and operational; move long-term strategy to [../PRODUCT.md](../PRODUCT.md) or backlog work to [../BACKLOG.md](../BACKLOG.md).
