# Activation

Owner: Product

Activation is the path from first session to first useful food-intent signal.

## Activation Moments

- First search with relevant results.
- First saved post or place.
- First review/post with dish or place context.
- First follow of a useful reviewer.
- First profile signal that reflects taste.

## Current Paths

- Guests can browse before auth.
- Auth prompts appear on meaningful actions.
- Sign-up collects profile details after credentials.
- Create flow prompts for location, ratings, cuisine, best dish, and dish tags.

## First 60 Seconds

- Target path: open app -> see food content -> search or save -> auth prompt only when intent is clear.
- Zero-follow feed should route to dishes nearby, saved-worthy posts, or taste guides.
- Suggested people should be framed by food taste and usefulness, not usernames alone.
- Profile setup should stay lightweight until it improves immediate discovery.

## Signals

- Search-to-save conversion.
- Guest prompt conversion on save, like, follow, comment, or create.
- Sign-up completion after credential entry.
- First post completion and first save completion.
- First food content seen before auth.
- First follow from a taste-guide or food-context prompt.
- Time from install/open to first save/search/follow.

## Gaps

- New users with no follows can still hit low-context social suggestions before clear food value.
- Search/feed empty states need to keep pointing to dishes, saves, nearby food, and taste guides.
- Activation analytics should measure first useful food-intent signal, not account creation alone.

## Guardrails

- Do not block browsing too early.
- Ask for profile/taste input only when it improves immediate utility.
- Keep onboarding lightweight until content density supports personalization.
