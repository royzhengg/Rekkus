# Design Docs

Design docs own Rekkus UI rules, reusable component guidance, interaction patterns, and copy guidance. Product behavior belongs in [../product](../product); execution work belongs in [../BACKLOG.md](../BACKLOG.md).

## Active Docs

- [DESIGN_SPEC.md](DESIGN_SPEC.md): design system, tokens, layout, and screen guidance
- [UI_LIBRARY.md](UI_LIBRARY.md): reusable UI primitives and component inventory
- [UX_Copywriting_Guide.md](UX_Copywriting_Guide.md): copy rules and tone guidance
- [DESIGN_SYSTEM.md](DESIGN_SYSTEM.md): design-system contract and surface patterns
- [TOKENS.md](TOKENS.md): color, spacing, radius, and typography token ownership
- [COMPONENT_PATTERNS.md](COMPONENT_PATTERNS.md): component placement and reuse rules
- [MOTION.md](MOTION.md): animation principles and guardrails
- [ACCESSIBILITY.md](ACCESSIBILITY.md): mobile accessibility checklist and priority flows
- [APPLE_HIG_REFERENCE.md](APPLE_HIG_REFERENCE.md): official Apple HIG applicability index and Rekkus implementation backlog map

## Governance

- Use existing tokens and components before introducing new patterns.
- Update design docs when visual rules, reusable components, popup/action-sheet patterns, accessibility rules, or copy conventions change.
- Avoid duplicating feature behavior here; link to the relevant product doc instead.
